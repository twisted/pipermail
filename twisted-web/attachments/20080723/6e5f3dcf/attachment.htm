</div></div>Hey Arun,<br>
<br>
The problem is that you&#39;re mixing two different patterns: an object<br>
store (pre-created hierarchy of parents/children) and dynamically<br>
generated children.<br>
<br>
The latter is the easiest case to talk about, since it&#39;s the classic<br>
HTTP GET where you pass all the arguments you need for any given<br>
request. If that&#39;s what you want to do, then your application needs to<br>
add query parameters to every link that it renders.<br>
<br>
The quickest way to do that with your example code and without<br>
changing anything would be to change Calendar.render to the following:<br>
<div class="Ih2E3d"><br>
 &nbsp; &nbsp;def render(self,request):<br>
 &nbsp; &nbsp; &nbsp; &nbsp;return &quot;&quot;&quot;<br>
 &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp;&lt;p&gt; The user is %s<br>
</div> &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp;&lt;a href=/calendar/month?user=%s&amp;pswd=%s&amp;server=%s&gt;The<br>
<div class="Ih2E3d"> &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp;link to the month is this &quot;&quot;&quot; % (<br>
</div> &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp;self.user[0], self.user[0], self.pswd[0], self.server[0])<br>
<br>
and remove your HomePage.__init__ method.<br>
<br>
The first option is what I&#39;d guess you were originally aiming for.<br>
This is a very different pattern: you want an object to be created<br>
with appropriate data stored on attributes, and you want to be able to<br>
refer to those objects and the values of their attributes in the<br>
future, not create them on the fly. There are probably some quick<br>
hacks that could demonstrate how to do this, but as soon as you tried<br>
to do anything remotely real with those hacks, you&#39;d be in trouble. If<br>
this is really what you want, then you should spend some time getting<br>
to know Axiom.<br>
<br>
Hope that helps. If not, feel free to ask more questions,<br>
