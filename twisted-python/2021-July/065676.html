In this example you're doing `python -m twisted.conch`, not `python -m twisted.conch.stdio`, which will not work anywhere.
Unfortunately, the Conch standard-IO library (and therefore the conch terminal client, and twisted.conch.stdio) doesn't work on Windows and never has: https://twistedmatrix.com/trac/ticket/2157 <https://twistedmatrix.com/trac/ticket/2157>.  However, you may be able to get it up and running with the Windows Subsystem for Linux, which I believe will present a Linux-like interface since as far as the application knows it's running on Linux: https://docs.microsoft.com/en-us/windows/wsl/about <https://docs.microsoft.com/en-us/windows/wsl/about>
