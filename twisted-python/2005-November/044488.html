Also, I should point out that implementing setUpClass using resources
will (probably -- I haven't benchmarked) make tests run faster.

Say you have a base:

class ComplexBase(unittest.TestCase):
  def setUpClass(self):
    """do complex stuff"""

  def tearDownClass(self):
    """do complex stuff"""

class A(ComplexBase):
  def test1(self):
    pass

class B(ComplexBase):
  def test2(self):
    pass

Then, with resources, the run order will be:
    setUpClass
    test1
    test2
    tearDownClass

As opposed to:
    setUpClass
    test1
    tearDownClass
    setUpClass
    test2
    tearDownClass
