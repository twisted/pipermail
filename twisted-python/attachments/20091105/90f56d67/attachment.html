Are you aware that this means your server will only be able to handle<BR>
one request at a time?  If so and that's what you're shooting for, then<BR>
you can fix this easily, by passing False to reactor.run.  This<BR>
disable's Twisted's own process support and lets things like<BR>
subprocess.call work (blockingly, of course).<BR>
>What I have now looks like (enormously simplified):<BR>
>class MyPythonServer:<BR>
><BR>
>    def serverMethod1(self, dummyPost, inputFilePath):<BR>
>        # Open inputFilePath, do stuff, save results to file2<BR>
>        subprocess.call(["command1", "file2", "file3"])<BR>
>        # Open file3, do stuff to the data, save results to file4<BR>
>        subprocess.call(["command2", "file4", "file5"])<BR>
>        # Open file5, convert data to a large string containing XML.<BR>
>        return xmlString<BR>
<BR>
Of course, you can still have methods that look like this if you use<BR>
things like inlineCallbacks (which you can, since you said you're using<BR>
Python 2.6).  That way you preserve the style you seem to prefer and<BR>
avoid blocking your server any time you need to launch a child process.<BR>
