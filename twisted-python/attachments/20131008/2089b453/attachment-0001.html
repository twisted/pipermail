    That's a good idea; I opened a ticket to document that.<br>
    <br>
    Short version:<br>
    <br>
    1. Maybe you're not writing a web app, but some other application or
    a generic library, in which case WSGI is irrelevant.<br>
    <br>
    2. Maybe you don't want to deploy on Twisted's WSGI container
    because you prefer some other WSGI container.<br>
    <br>
    Even if you decide to use Twisted as a WSGI container, or in fact
    any time you're using threads with Twisted where deferToThread is
    insufficient, Crochet's APIs for calling into Twisted from a thread
    are much nicer than blockingCallFromThread.<br>
