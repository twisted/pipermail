To run a program like apt, you must be using `reactor.spawnProcess´.  That<br>
means that you have two different protocol implementations.  One to talk<br>
to the SSH connection (eg, `SSHDemoProtocol´ in the O'Reilly example) and<br>
one to talk to the child process.<br>
<br>
If you want output from the child process to show up in the SSH client, then<br>
when the process protocol receives data from it, it should pass it on to the<br>
SSH protocol to write to the client.  Similarly, if you want input from the<br>
SSH client written to the stdin of the child process, when the SSH protocol<br>
receives data it should pass it on to the process protocol to write to the<br>
child process.<br>
<br>
You'll have to keep track of what state your application is in to do this<br>
correctly.  Input should only be handled this way when there is actually<br>
a child process running.<br>
