INTRODUCTION<br>
<br>
As some of you may know, the PSF has put together a team of volunteers<br>
who are maintaining a central Python events calendar. We currently have<br>
two calendars in place:<br>
<br>
 * Python Events Calendar - meant for conferences and larger gatherings<br>
   focusing on Python or a related technology (in whole or in part)<br>
<br>
 * Python User Group Calendar - meant for user group events and other<br>
   smaller local events<br>
<br>
The calendars are displayed on <a href="http://pycon.org/" target="_blank">http://pycon.org/</a> and in a smaller<br>
version in the sidebar of the <a href="http://python.org/" target="_blank">http://python.org/</a> website.<br>
<br>
You can subscribe to the calendars using iCal and RSS feeds and<br>
also embed the calendar widgets on your sites. Please see our wiki<br>
page for details:<br>
<br>
    <a href="https://wiki.python.org/moin/PythonEventsCalendar" target="_blank">https://wiki.python.org/moin/PythonEventsCalendar</a><br>
<br>
The calendars are open to the world-wide Python community, so you<br>
can have local user group events, as well as regional and international<br>
conference events added to the calendars.<br>
<br>
________________________________________________________________________<br>
NEWS<br>
<br>
We have now created a Twitter feed for the calendars, which you can<br>
follow to get updates on all newly added events:<br>
<br>
    <a href="https://twitter.com/pythonevents" target="_blank">https://twitter.com/pythonevents</a><br>
<br>
The tweets come with links to the event listings, which you can<br>
add to your own Google calendars with a single click.<br>
<br>
________________________________________________________________________<br>
ADDING EVENTS<br>
<br>
If you want to have entries added to those calendars, please write<br>
to <a href="mailto:events@python.org">events@python.org</a> and include the following information:<br>
<br>
 * Name of the event<br>
 * Type of the event (conference, bar camp, user group, etc)<br>
 * Focus on Python and approximate size<br>
 * URL<br>
 * Location and country<br>
 * Date and time (if relevant)<br>
<br>
For recurring events, please also include a description of the<br>
recurrence in a way that's compatible and supported by Google<br>
calendars.<br>
