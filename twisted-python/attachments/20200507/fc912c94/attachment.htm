Looking at the docs I found that Generators-based coroutines, something<br>
that twisted's inline callbacks depend a lot, will be deprecated soon by<br>
the time python 3.10 is released.<br>
<br>
<a href="https://docs.python.org/3/library/asyncio-task.html#generator-based-coroutines" rel="noreferrer" target="_blank">https://docs.python.org/3/library/asyncio-task.html#generator-based-coroutines</a><br>
<br>
<br>
* NoteSupport for generator-based coroutines is deprecated and is scheduled<br>
for removal in Python 3.10.*<br>
<br>
This is a problem since any future python release will come without the<br>
generator-based coroutines that twisted needs, so any higher version of<br>
python 3.10 will, for the most part, completely broke a lot of twisted code<br>
that already exists.<br>
How will then twisted handle this deprecation?<br>
