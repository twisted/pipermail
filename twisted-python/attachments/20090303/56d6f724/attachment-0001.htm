Assuming that you&#39;ve made sure you aren&#39;t leaking any connections, you are still faced with the problem of that the select module on Windows is limited to 512 file descriptors. Your options:<br>
<br>
1. Switch to Linux, where you can use other reactors with a much higher limit, e.g. epoll.<br>
2. Use the IOCP reactor on Windows, which isn&#39;t as well tested and is missing some features, in particular SSL support.<br>
3. Recompile the select module on Windows with a higher limit; basically it&#39;s some C constant you redefine, it&#39;s probably right at the top of the file. You just get the Python tarball, and rebuild the appropriate module (or everything and just extract the appropriate PYD).<br>
