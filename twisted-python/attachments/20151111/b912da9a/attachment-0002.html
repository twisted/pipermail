General rules by Glyph as they were sent to me :)<br>
<br>
Development can't stop, the website can't go down, and we can't lose<br>
any data. If you have a plan that migrates absolutely everything to<br>
github, including all of our issues to github issues, and all of our<br>
review queue stuff to github PRs, that is fine.<br>
<br>
You cannot, however, just push everything to github and delete the<br>
ticket database and all the outstanding branches and just say "okay<br>
everybody file github issues now". There has to be clear communication<br>
about what a developer who shows up on any given day to work on a<br>
Twisted ticket should do.<br>
<br>
Current requirements from Glyph:<br>
<br>
- be able to accept PRs on <a href="http://github.com" rel="noreferrer" target="_blank">github.com</a><br>
- host code primarily on github<br>
- make sure all the same committers still have access (at least active ones)<br>
- make sure the website doesn't go down<br>
- break as little functionality as possible (kenaan, highscores, etc)<br>
- communicate clearly to contributors what they have to do in order to<br>
work on Twisted in every step of the process<br>
<br>

<br>
