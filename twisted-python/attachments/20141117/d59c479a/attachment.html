<body><div>Looking at the code, <span style="font-family: menlo, consolas, "courier new", monospace, sans-serif;" class="font">_SignalReactorMixin.mainLoop()</span> takes the output of <span style="font-family: menlo, consolas, "courier new", monospace, sans-serif;" class="font">ReactorBase.timeout()</span> and passes it to <span style="font-family: menlo, consolas, "courier new", monospace, sans-serif;" class="font">doIteration()</span>. The implementation of <span style="font-family: menlo, consolas, "courier new", monospace, sans-serif;" class="font">doIteration()</span> in <span style="font-family: menlo, consolas, "courier new", monospace, sans-serif;" class="font">PollReactor</span> passes the value straight through to <span style="font-family: menlo, consolas, "courier new", monospace, sans-serif;" class="font">poll()</span>. If there is any kind of timed call pending (which there is), the timeout value counts down. Eventually it gets very close to zero. If there is a burst of AMP activity around that time, there's a good chance that the one of the <span style="font-family: menlo, consolas, "courier new", monospace, sans-serif;" class="font">poll()</span> calls will time out.<br></div>
