Hello again, and welcome to another update on the Twisted Lore to Sphinx conversion project.<br><br>Current Sphinx results are in the usual place at: <a href="http://twistedsphinx.funsize.net/">http://twistedsphinx.funsize.net/</a><br>
<br>Here&#39;s what&#39;s new:<br><br>- The lore2sphinx conversion script is now up on bitbucket.  You can find it here: <a href="http://bitbucket.org/khorn/lore2sphinx/">http://bitbucket.org/khorn/lore2sphinx/</a> .<br>  Special thanks to Steve Steiner (ssteinerX) for helping out a bitbucket newbie.<br>
<br>- lore2sphinx now reads input and output directories from a config file.  The config file can have one or more &quot;profiles&quot;, each of which contains config info for a set of documentation. At present, the script just uses the default profile (set in the config file), but I&#39;ll add a command line switch to specify the profile Real Soon Now (tm), which will make it easy to use a single config file for multiple projects (e.g. twisted, nevow, mantissa, etc.)<br>
<br>- lore2sphinx has also been refactored a bit, which solves a lot of whitespace handling issues. The Sphinx build now gives fewer than 100 errors (currently 65, started off in the 500-600 range)!  Hooray!  Still, not all of the output is fully correct, and some tags (notably tables) are still unhandled.<br>
<br>- I still haven&#39;t started any work on a theme...I&#39;ll get to it eventually, unless someone else does first.  Any volunteers?<br><br>- Divmod docs: new config file support is a step in this direction, look for something soon...<br>
<br>- PyOpenSSL docs: kind of a sideline to this so I haven&#39;t really moved forward on this yet, but I will eventually<br><br><br>If anyone out there would like to help out, I can certainly use the assistance.  Lore2sphinx could probably use some extra eyeballs and brains looking at it, thought the basic structure is pretty much laid out.  Any ideas/comments/criticisms welcome.<br>
