I was wondering if anybody subscribed to this list knew the process for
writing an RFC, and if so, thought it was worth the time and effort to do
so for the V3 protocol ...

It might be nice to write a more generalized version of the TR protocol
stuff, since it generally seems to be a highly useful protocol and might
be good for other things as well ... it has the same advantage of HTML in
that you can afford to ignore parts of the message if you don't understand
what they're about... so any incrementally developed system might be able
to benefit from it.

Also, it might be nice to have a crypted version of the same thing.
