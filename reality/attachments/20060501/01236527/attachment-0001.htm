Jp and I will be working on &quot;OBJECT PROXIES&quot; starting tonight. These are meant to solve the problem of dynamically changing the behavior based on intermediary game objects. FOR EXAMPLE: A sound-proof glass window should let light through but not sound. This would be implemented by capturing events which traverse the glass window, and removing aurial concepts from that event. MAYBE. We don't actually know how we're going to do this, but we'll write some unit tests and see where that takes us.
