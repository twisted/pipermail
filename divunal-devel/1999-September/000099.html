Attached are some files that are the beginnings of the universal java
Makefile system that I want to include as part of TR -- obviously, they're
not finished yet.

I say "universal" because you'll note that the system includes hooks for
RMI (not something that's going to be finding its way into TR for quite a
while). It's a simple build process designed to work with any
properly-organized Java class-library...

The nice thing about these makefiles is that they will work "out of the
box" with Emacs, XEmacs, or Vim, using either Javac or Jikes.  (Also, with
other tools, like the ones Mozilla uses to do all the funky stuff they do
with their CVS tree)

I'm not happy about the number of files this requires though, and also
about the fact that the build process depends upon Perl.  Could anyone
re-implement the functionality of MakeMakeFiles.pl and JavaDep.pl in "100%
pure make"?

If not, it's not a big deal... I think we can reasonably assume that most
people who have 'make' installed also have some version of perl...
